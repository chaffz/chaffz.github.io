水壶问题

水壶问题，从上面的图中，其实可以总结出以下几点：

  • 两个水壶不断改变水量，求满足当前水量x+y=Target是否成立

  • 状态，[x, y]组成的双值对象定义一种状态，即两个水壶的当前水量

  • 状态转移过程之间的动作可分为3大类，6个小动：

    1. 装满：装满A、装满B

    2. 清空：清空A、清空B

    3. 转移：A倒入B、B倒入A

推荐解法：裴蜀定理（贝祖定理）

裴蜀定理（或贝祖定理）得名于法国数学家艾蒂安·裴蜀，说明了对任何整数a、b和它们的最大公约数d，关于未知数x和y的线性不定方程（称为裴蜀等式）：若a,b是整数,且gcd(a,b)=d，那么对于任意的整数x,y,ax+by都一定是d的倍数，特别地，一定存在整数x,y，使ax+by=d成立。

它的一个重要推论是：a,b互质的充分必要条件是存在整数x,y使ax+by=1.

此定理对应成代码逻辑则是：

x
// B 表示小水壶容量，A 表示大水壶容量
while(sum) {
  if (sum == target) return true;
  // 如果当前值大于目标值，则减去小的
  // 可对应定理中的减 n*B
  if (sum > target) sum - B;
  
  // 如果当前值大于目标值，则减去小的
  // 可对应定理中的加 m*A
  sum += A;
}

所以代码中就实现了 m∗A−n∗B 的过程。

其它解法：结合上述状态描述，那么有如下状态：

  1. A清空：(0, y)

  2. B清空：(x, 0)

  3. 装满A： (capX, y)

  4. 装满B： (x, capY)

  5. A倒入B：(max(x−capY+y,0),min(x+y,capY))

  6. B倒入A：(min(x+y,capX),max(y−capX+x,0))

所以使用动态规划还是BFS、DFS均可。
