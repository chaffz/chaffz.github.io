kubectl 使用

kubectl 是 Kubernetes 自带的客户端，可以用它来直接操作 Kubernetes。

使用格式有两种：

get

显示一个或多个资源

describe

显示资源详情

create

从文件或标准输入创建资源

update

从文件或标准输入更新资源

delete

通过文件名、标准输入、资源名或者 label selector 删除资源

log

输出 pod 中一个容器的日志

rolling-update

对指定的 replication controller 执行滚动升级

exec

在容器内部执行命令

port-forward

将本地端口转发到Pod

proxy

为 Kubernetes API server 启动代理服务器

run

在集群中使用指定镜像启动容器

expose

将 replication controller service 或 pod 暴露为新的 kubernetes service

label

更新资源的 label

config

修改 kubernetes 配置文件

cluster-info

显示集群信息

api-versions

以 "组/版本" 的格式输出服务端支持的 API 版本

version

输出服务端和客户端的版本信息

help

显示各个命令的帮助信息
